Comment résoudre les problèmes d'initialisation OpenPGP

Il y a plusieurs raisons à l'échec d'initialisation d'OpenPGP. Les plus communes sont décrites ci-dessous ; pour plus d'information consulter le support Enigmail.

GnuPG introuvable

Pour qu'OpenPGP fonctionne, le logiciel GnuPG doit être installé. Si GnuPG est introuvable, vérifiez que le programme eécutable gpg.exe (sous Windows ; gpg pour les autres plate-formes) est intallé sur votre ordinateur. Si GnuPG est installé et qu'OpenPGP ne peut pas le trouver, il faut préciser manuellement le chemin d'accès à GnuPG dans les options d'OpenPGP (menu OpenPGP > Préférences)

Échec d'intialisation Enigmime

OpenPGP ne fonctionne que s'il a été compilé selon le même environnemnt que Thunderbird ou SeaMonkey. Cela veut dire que vous ne pouvez utiliser les versions officielles d'Enigmail que si vous utilisez les versions officielles de Thunderbird ou SeaMonkey fournies par mozilla.org.

Si vous utilisez des versions de Thunderbird ou SeaMonkey d'autres sources (ex. : le fournisseur de votre distribution Linux), ou si vous avez compilé l'application vous même, vous devrez utiliser une version d'Enigmail provenant de la même source. Pour compiler Enigmail consultez la section code source sur le site Enigmail. Merci de ne pas rapporter de bug concernant ce problème ; il n'y a pas de solution.

De l'aide complémentaire est disponible sur le site de support Enigmail.
