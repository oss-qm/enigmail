Como Resolver Problemas com a Inicialização do OpenPGP

Várias razões podem fazer falhar a inicialização do OpenpGP. As mais comuns são descritas abaixo& para mais informação, visite por favor a página de suporte do Enigmail.

Não foi possível encontrar o GnuPG

Para que o OpenPGP funcione, é necessário que a ferramenta GnuPG esteja instalada. Se o GnuPG não foi encontrada, assegure-se que o programa gpg.exe (no Windows& gpg nas outras plataformas) está instalado no seu computador. Se o GnuPG está instalado e o OpenPGP não o consegue encontrar, necessita de configurar manualmente o caminho para o GnuPG nas Preferências do OpenPGP (menu OpenPGP > Preferências)

O Enigmime não consegue inicializar-se

O OpenPGP funciona apenas se for compilado com o mesmo tipo de sistema que o Thunderbird ou Seamonkey tiverem sido compilados. Isto significa que apenas pode usar as versões oficiais do Enigmail se também estiver a usar as versões oficiais do Thunderbird ou Seamonkey fornecidas em mozilla.org.

Se usa uma versão do Thunderbird ou Seamonkey provenientes de outra fonte (ex: o fornecedor da sua distribuição de Linux), ou se foi o próprio a compilar a aplicação, deve usar uma versão do Enigmail compilada pela mesma fonte ou por si. Para compilar o Enigmail, consulte a secção Source Code na página web do Enigmail. Por favor, não reporte nenhum erro relativo a um problema deste tipo, pois não pode ser solucionado.

Pode encontrar mais ajuda no Sítio Web de Suporte ao Enigmail.
