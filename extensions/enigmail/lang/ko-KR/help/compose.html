메시지 서명

서명된 메시지 보내기를 켜거나 끕니다. 서명이 실패하면 사용자에게 알려줍니다.

메시지 암호화

보내기 전, 모든 받는 사람에게 암호화된 메시지 보내기를 켜거나 끕니다. 암호화가 실패하면 사용자에게 알려줍니다.

설정 → 키 선택 탭에서 키 선택 기준으로,

미리 정의된 규칙이 선택되어 있으면 받는 사람별 규칙에 따라 받는 사람 공개키를 선택합니다. 규칙이 없는 경우 사용자에게 새로운 규칙을 만들지 여부를 물어봅니다.

전자 메일 주소가 선택되어 있으면 전자 메일 주소를 체크하여 공개키를 선택합니다. 공개키를 가지고 있지 않은 사람에게 메시지를 보내기 위해 받는 사람 목록에 전자 메일 주소를 입력한 경우, 수동 선택하거나 키서버에서 키를 가져올 수 있는 키 목록 대화 상자가 나타납니다.

규칙과 전자 메일 주소가 선택되어 있으면 일단 규칙을 적용한 후, 전자 메일 주소로 키를 선택합니다. 공개키를 가지고 있지 않은 사람에게 메시지를 보내기 위해 받는 사람 목록에 전자 메일 주소를 입력한 경우, 수동 선택하거나 키서버에서 키를 가져올 수 있는 키 목록 대화 상자가 나타납니다.

항상 사용자가 선택 이 선택되어 있으면 암호화된 메시지를 보내려할 때마다 키 목록 대화 상자가 나타납니다.

설정 → 키 선택 탭에서 키 선택을 수동으로는 하지 않음이 선택되어 있는 상태에서 공개키를 가지고 있지 않은 받는 사람에게 메시지를 보내기 위해 받는 사람 목록에 전자 메일 주소를 입력한 경우, 해당 메시지는 암호화되지 않은 상태로 보내집니다

이 메시지에 PGP/MIME을 사용

이 메시지에 PGP/MIME 사용 여부를 켜고 끕니다.

받는 사람이 PGP/MIME 형식의 메일을 읽을 수 있으면, 이 방식을 사용해야합니다.

이 기능은 도구 → 계정 → OpenPGP 보안 → 메시지 작성 기본 옵션 그룹에서 항상 PGP/MIME을 사용 여부를 활성화/비활성화 할 수 있습니다.

받는 사람별 규칙을 무시

받는 사람별 규칙을 적용하지 않습니다. 설정 → 키 선택 탭에서 키 선택 기준으로 미리 정의된 규칙이 선택되어 있는 경우, 키 선택을 할 수 없기 때문에 암호화한 메시지를 보내려고하거나 암호화하여 저장하려고 할 때 실패하며, 규칙을 만들 것인지도 묻지 않습니다.

암호화 실행 취소

실재로 메일을 보낼 때 POP 서버가 요청 수락을 하지않아 실패하면 Enigmail은 그런 일이 일어났는지 모릅니다. 그래서 암호화된 메시지가 계속 작성 윈도우에 표시됩니다. 이 메뉴를 선택하면 원래 텍스트가 들어있던 상태로 작성 윈도우를 되돌려서 암호화/서명을 실행 취소하게 됩니다.
버그를 회피하기 위한 일시적인 방법으로서, 이 옵션은 암호화된 메시지에 대해 회신할 때 인용된 텍스트를 복호화하는데 사용할 수도 있습니다. Enigmail은 인용된 메시지를 자동적으로 복호화 해야하지만, 어떤 문제로인해 이 작업이 실패하면 이 메뉴를 사용해서 강제로 복호화할 수 있습니다.

내 공개키 첨부

내 공개키를 첨부파일 형식으로 메시지에 첨부합니다.

공개키 첨부

ASCII 보호형 공개키 텍스트를 작성 윈도우의 현재 커서 위치에 붙여넣기 합니다. 붙여넣기할 키의 전자 메일 주소를 물어보기위해 프롬프트를 출력하게됩니다. 이렇게 보내진 키는 받는 쪽에 설치된 Enigmail에서 인식됩니다. 붙여넣기 후에도, 필요에 따라 메일을 서명/암호화할 것인지를 선택할 수 있습니다. 메시지에는 한개 이상의 키는 붙여넣지 마시기 바랍니다. 필요하면 프롬프트가 출력되었을 때, 콤마나 공백문자로 분리하여 여러 메일 주소를 지정하면 한개 이상의 키를 보낼 수 있습니다.
